# VPN Source Management Skill

## Когда использовать

Этот skill применяется для задач:
- import / upload VPN URI;
- валидация протоколов;
- обновление/замена существующих источников;
- фильтрация и выборка по параметрам запроса;
- проектирование таблиц и репозиториев каталога VPN-источников.

## Главная идея

Исходные VPN-ссылки — это отдельный каталог доменных объектов, а не просто строковый список в таблице подписок.

## Обязательный алгоритм

1. Определи, чем является операция:
   - загрузка нового источника;
   - массовый импорт;
   - замена существующего источника;
   - выборка источников для сборки подписки;
   - управление статусом / приоритетом / тегами.

2. Покажи минимум 2 варианта хранения метаданных:
   - всё в одной таблице с простыми полями;
   - отдельные связи для tags / sets / rules.

3. Выбери вариант, подходящий текущему этапу.

## Правила моделирования

По умолчанию в `vpn_sources` нужно иметь:
- protocol;
- безопасно сохранённый raw URI;
- status;
- priority;
- display_name;
- метаданные, нужные для выбора;
- timestamps.

Если выборка становится сложной и бизнес управляет ею как данными, а не кодом, предлагай:
- `vpn_source_tags`;
- `vpn_source_sets`;
- `source_selection_rules`.

## Валидация

- Валидируй схему URI и базовую структуру.
- Не обещай полную совместимость протокола, если реализация валидирует только часть правил.
- Храни сырое значение безопасно.
- Не логируй полные URI.

## Обязательные вопросы к задаче

- Нужен ли soft delete?
- Нужна ли ротация/замена активных источников?
- Нужна ли история изменений?
- Должна ли выданная подписка фиксировать snapshot источников или достаточно ссылок на текущие записи?

Если задача этого не уточняет, по умолчанию рекомендуй хранить snapshot/fingerprint на уровне issued subscription items.
