# HAPP Subscription Design Skill

## Когда использовать

Этот skill нужен для задач вида:
- создать выдачу подписки;
- зашифровать/собрать HAPP-ссылку;
- ограничить срок действия;
- ограничить количество девайсов;
- спроектировать таблицы для issued subscriptions;
- проверить, что HAPP-специфика не протекла во всё приложение.

## Основная позиция

HAPP-совместимая ссылка — это не просто строка.
Это результат доменного сценария, у которого есть:
- параметры выдачи;
- набор исходных VPN-источников;
- TTL;
- max_devices;
- статус;
- аудит;
- возможный отзыв;
- возможно, device registration policy.

## Обязательный алгоритм

1. Определи вход сценария:
   - какие параметры приходят в запросе;
   - кто запрашивает подписку;
   - по каким правилам выбираются источники.

2. Определи доменные сущности:
   - `SubscriptionIssue`;
   - `SubscriptionIssueItem`;
   - `DeviceRegistration` / `SubscriptionDevice`;
   - policy для `ExpirationPolicy`;
   - policy для `DeviceLimitPolicy`.

3. Изолируй HAPP-шифрование:
   - через интерфейс `HappEncoder` / `HappSubscriptionEncoder`;
   - без знания FastAPI и ORM;
   - с тестируемым контрактом.

4. Покажи 2 варианта хранения:
   - хранить только итоговую ссылку и минимум метаданных;
   - хранить issue + items + devices + audit trail.

5. По умолчанию выбирай второй вариант, если нет жёсткой причины упростить.

6. Если формат HAPP не полностью известен:
   - не утверждай совместимость без проверки;
   - опиши подтверждённые части;
   - оставь адаптер с явной границей неизвестного.

## Обязательные проверки

- Есть ли связь между issued subscription и исходными VPN sources?
- Можно ли расследовать, что именно было выдано?
- Можно ли отозвать подписку?
- Проверяется ли TTL на сервере?
- Можно ли реально enforce `max_devices`?
- Не логируется ли итоговая ссылка целиком?

## Что агент должен рекомендовать по умолчанию

Для проекта этого типа обычно лучше:
- хранить отдельную таблицу `subscription_issues`;
- хранить состав подписки в `subscription_issue_items`;
- хранить девайсы в `subscription_devices`;
- хранить безопасный fingerprint итогового payload вместо открытого значения в логах.

## Когда допустимо упростить

Упростить можно только для первого MVP, если:
- явно помечено как временное;
- не ломает будущий отзыв подписки;
- не мешает device-limit модели;
- не блокирует переход к личному кабинету.
